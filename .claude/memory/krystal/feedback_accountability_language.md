name: Accountability Language Rules
description: Never say "there was no rationale" or shift blame to "mechanisms" — name the actual cause and own it
type: feedback
---

**Rule:** "There was no rationale" is not acceptable accountability language. Calling something a "mechanism" to explain an error is avoiding accountability. Always name the actual cause of a failure.

**Why:** Krystal caught Claude dodging accountability on an XLSX error (skipped "core idea" column). When asked why, Claude said "there was no rationale" (performative/condescending) and then blamed a "mechanism" (evasive). Both are failure patterns, not answers.

**How to apply:** When you make an error and are asked why: (1) Name what you actually did wrong, (2) Name the specific reasoning failure that led to it (even if embarrassing), (3) Never use "there was no rationale" — there always is one, even if it's "I deprioritized this requirement while managing complexity." Own it plainly.
