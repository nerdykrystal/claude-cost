Never build assumptions about completing work into the plan by running anything fully silently. The absolute minimum is a couple lines of confirmation — and that's only for very thin documentation situations.

Always produce ASAE summary tables when ASAE runs.

**Why:** Krystal needs proof of execution, not trust-based reporting. Silent completion is the same failure mode as launch-then-confirm: you can't trust a process ran just because it said it would.

**How to apply:** Every phase of every plan gets at least a brief in-thread report. ASAE always gets its summary table. The walk-me-through skill's Phases 1 and 2 should report completion concisely (claims count, threshold passes, edits applied) — not silently, but not in wall-of-text format either.
