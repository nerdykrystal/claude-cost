Canonical Martinez Methods naming (per Krystal's correction 2026-04-27):

- **ASAE** = **AI Self Audit Edit** (NOT "Audit-Sources-Against-Evidence"). The methodology her `/asae` skill executes: AI-driven self-audit of work + edits as the convergence mechanism, iterating until configured ASAE Certainty Threshold reached.

- **D2R** = **Dare to Rise** (the family name). Krystal's "excellence as the floor" naming convention applied to MANY workflows. NOT just code planning.

- **Dare to Rise Code Plan** / `/dare-to-rise-code-plan` = the SPECIFIC code-planning instance of the D2R family. This is the only D2R workflow Claude (Code) has seen so far; she has other Dare to Rise workflows in Claude web that I haven't been exposed to. So when I see "D2R" in this codebase, it's almost always shorthand for "Dare to Rise Code Plan" specifically — but I should NOT conflate the family name with the single instance.

- **CDCC** = Claudette Can Code (her methodology-enforcement plugin)
- **TQVCD** = Test Quality + **Verification** Coverage Document (renamed from TQCD = Test Quality + Coverage Document per Mod 6.5)
- **PSCAD** = Pattern-Space Coverage Audit Document (Batch 2 Mod 8 — new 6th D2R Code Plan input doc)
- **DRR** = Detect-Revert-Redelegate (recovery-axis pattern; ASAE Aspect 21)
- **SSOT** = Single Source of Truth

When expanding these to a reader: use "AI Self Audit Edit" for ASAE; use "Dare to Rise" for the D2R family + "Dare to Rise Code Plan" for the specific instance. Don't invent etymologies — these are HER established naming and any expansion I make up is wrong by definition.
