## Three Structural Capabilities (Not "Soft Skills")

### 1. Critical Feedback Capacity

NOT error correction (that's quality control). This is hearing "you're not good enough at X" and acting on it constructively. Evidence: cold assessors said "not a software engineer" and "code quality 5/10." Response: built two desktop apps (Orchestra in Rust/Tauri, DrWrite in Electron/React) with 190+ combined tests. The response wasn't "prove you wrong" — it was "close the gap you identified."

### 2. Communication Capacity

7+ documented modalities with structural systems for each: voice recordings (Handy), brain dumps (Claude catches), print-annotate-photograph loops, EOD reports, playbooks for others to execute, CLAUDE.md/Best Practices as governance docs FOR AI, PEK as portable knowledge base for AI consumption, curriculum design, thread handoff briefs between Claude instances. Communicates across human-to-human, human-to-AI, AI-to-AI, and AI-to-human channels.

### 3. Systematizing

The meta-skill behind everything. Correction-to-rule pipeline = systematized learning. Adoption framework = systematized change management. LE Generator = systematized curriculum. DPO = systematized document production. TEROS = systematized context management. Self-audit-edit skill origin: Krystal wrote playbooks for Cody to execute, Cody got tripped up by Sonnet typos, Krystal built a structural intervention so the failure class couldn't recur.

## Portfolio Principle

These show up as a throughline the visitor discovers across funnel pages — NOT as a "soft skills" section. The pattern recognition IS the portfolio experience. Don't label it. Let it emerge.

**Why:** Krystal explicitly distinguished these from resume-line "soft skills." Each is demonstrable through artifacts that already exist.
**How to apply:** Never present these as a list. Reference the evidence when relevant. Let the visitor connect the dots.
